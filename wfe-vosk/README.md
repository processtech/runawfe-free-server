💡 RunaWFE автоматически объединяет все settingsList.xml из META-INF/resources/ всех модулей при запуске.

wfe-web будет зависеть от wfe-speech-api (всегда, compile scope).

wfe-vosk будет реализовывать этот интерфейс и зависеть от wfe-speech-api (compile).

wfe-web компилируется всегда (интерфейс есть), а реализация подтягивается только при активном профиле.

EJB-контейнер сам управляет жизненным циклом бина (создаёт экземпляры, пулирует их). 

wfe-speech-api → контракт
wfe-vosk → реализация
wfe-web → клиент (servlet + UI)

в яндекс браузере распознает спомощью браузера в фаерфоксе через модель.
